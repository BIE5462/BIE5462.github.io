hexo博客部署

杨春波

2020-09-25

技术笔记

这是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

GitHub 是通过 Git 进行版本控制的软件源代码托管服务,使用github pages服务搭建的博客访问速度快，免费方便，不需要服务器不需要后台，可以随意绑定自己的域名，不仔细博客内容还可以轻松打包、转移、发布到其它平台。

准备工作

  • 注册github账号 https://github.com/
  • 安装node.js、npm http://nodejs.cn/
  • 安装git for windows https://git-scm.com/downloads

文件夹目录

  • _config.yml文件用于存放网站的配置信息，你可以在此配置大部分的参数.
  • scaffolds是存放模板的文件夹，当新建文章时，Hexo会根据scaffold来建立文件.
  • source是资源文件夹，用于存放用户资源.
  • themes是主题文件夹，存放博客主题，Hexo 会根据主题来生成静态页面。

使用hexo写博客

原理

由于github pages存放的都是静态文件，博客存放的不只是文章内容，还有文章列表、分类、标签、翻页等动态内容，假如每次写完一篇文章都要手动更新博文目录和相关链接信息，相信谁都会疯掉，所以hexo所做的就是将这些md文件都放在本地，每次写完文章后调用写好的命令来批量完成相关页面的生成，然后再将有改动的页面提交到github。

修改主题

1.找到自己喜欢的主题https://hexo.io/themes/
2.将文件解压到站点的 themes 目录下， 并将解压后的文件夹名称更改为material-x （本例使用material-x主题）
3.打开站点配置文件 _config.yml，找到 theme 字段，并将其值更改为 material-x。
4.在Git Bash终端执行命令hexo s，在浏览器中访问http://localhost:4000/，可以看到主题样式。
5.如果出现异常，可以先执行hexo clean来清理一下public的内容，然后再来重新生成和发布。
6.主题配置参考说明文档https://xaoxuu.com/wiki/material-x/
